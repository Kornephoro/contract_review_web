# 智审法务 PRD（产品功能规划）

## 1. 产品概述
智审法务是一个本地运行的 AI 合同审查工作台，面向中文商业合同场景，核心目标是把“风险识别”变成“可执行修改”，让用户可以从审查、追问到导出在同一流程内完成。

## 2. 要解决的问题
- 合同审查效率低，人工逐条阅读耗时长。
- 审查结果不结构化，沟通成本高。
- 很多建议停留在“方向描述”，难以直接回写合同。
- 扫描件合同处理困难，文本抽取链路不稳定。

## 3. 用户角色
- 法务人员：日常审查主力，要求稳定、可追踪、可导出。
- 律师/律师助理：关注法律依据、条款可执行性、谈判策略。
- 业务负责人：关注风险级别、影响范围、是否可签。

## 4. 设计原则
- 输出结构化：所有审查结果可被系统读取、筛选、导出。
- 建议可落地：优先生成可直接替换回原文的条款文本。
- 过程可解释：每条风险都能定位到原文，且包含风险说明。
- 操作可闭环：支持应用建议、追问细化、导出交付件。

## 5. 核心流程
1. 用户输入合同（上传文件或粘贴文本）。
2. 系统抽取正文（文本提取或 OCR）。
3. 用户选择审查参数（深度、立场、模板）。
4. 系统输出结构化风险结果。
5. 用户在工作台处理风险（筛选、查看原文、应用建议）。
6. 用户对单条风险或整份合同继续追问。
7. 用户导出审查报告或修订文档。

## 6. 功能规划（详细）

### 6.1 合同输入中心
目标：统一多来源合同输入，保证后续链路稳定。

功能点：
- 文件上传：支持 `.docx`、`.pdf`、`.png`、`.jpg`、`.jpeg`。
- 文本输入：支持直接粘贴合同正文。
- 输入校验：空文本、异常文件、解析失败时给出明确提示。
- OCR 分流：文本型 PDF 走文本提取，扫描型文件自动走 OCR。

交互要求：
- 上传后立即反馈识别状态。
- OCR 未初始化时，给出可执行命令提示，不出现静默失败。

### 6.2 审查参数配置
目标：让同一份合同能适配不同审查场景。

功能点：
- 审查深度：快速、标准、深度。
- 审查立场：中立、委托方、相对方。
- 审校模板：按合同类型加载专项重点（含内置模板和自定义模板）。

交互要求：
- 参数改变后，下一次审查立即生效。
- 模板可查看详细内容，避免“黑盒审查”。

### 6.3 结构化审查结果
目标：输出可展示、可筛选、可导出的统一数据。

输出结构：
- 合同类型判断
- 合同概览（参与方、金额、期限、签署日期、适用法律、摘要）
- 风险列表（等级、维度、原文、问题、建议、法律依据）

规则要求：
- 风险维度固定为：法律合规、风险防控、条款完善、利益保护。
- `original` 必须来自合同原文，便于定位。
- `suggestion` 优先为可直接替换条款。

### 6.4 风险工作台
目标：把“读报告”升级为“处理风险”。

布局规划：
- 左侧：风险卡片区（筛选、排序、选中、应用按钮）。
- 中部：合同正文区（高亮锚点、已应用状态展示）。
- 右侧：风险追问区（围绕单条风险连续对话）。

能力要求：
- 按风险维度筛选。
- 按风险等级排序。
- 支持单条应用和一键应用全部可替换建议。
- 支持撤销应用，恢复原文。

### 6.5 追问与对话
目标：覆盖“风险细化”和“全局问答”两类场景。

功能点：
- 风险追问：围绕当前风险的原文、问题和建议继续提问。
- 上下文对话：基于合同全文和最新审查结果进行问答。
- 对话记录管理：支持清空、重置、切换追问对象。

### 6.6 模板中心
目标：沉淀审查策略，提高复用效率。

功能点：
- 内置模板：覆盖通用、买卖、劳动、租赁、服务、借款、保密、建设工程。
- 自定义模板：支持创建、编辑、删除。
- 模板绑定：支持绑定合同类型，减少人工切换成本。
- 模板回退：内置模板支持恢复默认。

### 6.7 导出中心
目标：直接产出可交付结果。

导出类型：
- 审查报告导出：HTML（概览 + 风险摘要 + 逐条明细）。
- 修订文档导出：
  - 原始为 `.docx` 时，优先做内容替换并尽量保留格式。
  - 非 `.docx` 时，提供 `.txt` 与普通 `.docx`。

### 6.8 配置中心
目标：在同一界面支持多模型和工具扩展。

功能点：
- 模型供应商配置（OpenAI 兼容、Anthropic 网关、Ollama）。
- API 参数配置（Key、Base URL、Model）。
- MCP 开关与提示说明（可选接入外部检索能力）。
- 本地配置持久化（下次启动保留配置与模板）。

## 7. 关键数据对象

### 7.1 风险对象
- `level`：高风险/中风险/低风险
- `dimension`：法律合规/风险防控/条款完善/利益保护
- `original`：原文摘录
- `issue`：风险说明
- `suggestion`：建议条款
- `legal_basis`：法律依据

### 7.2 工作台状态对象
- `review_snapshot`：当前审查快照
- `applied_risks`：已应用风险索引
- `focus_risk_idx`：当前追问风险索引

## 8. 非功能要求
- 环境：Windows + Python 3.11+。
- 可用性：首次使用可按提示完成 OCR 初始化。
- 稳定性：关键错误必须可视化提示，避免“点击无反应”。
- 可维护性：模板与参数本地持久化，重启后可恢复。

## 9. 范围边界
- 本产品用于审查辅助，不提供正式法律意见。
- 暂不覆盖多人协同审批、权限体系、云端案件库。
- 暂不做移动端独立应用。

## 10. 后续功能池
- 长合同分片审查与统一汇总。
- 审查结果版本对比（修订前后 diff）。
- 风险处置清单（责任人、状态、截止时间）。
- 审查日志与操作审计。
- 行业模板扩展与模板分享机制。
